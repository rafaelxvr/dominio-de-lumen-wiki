## Project Overview

**Domínio de Lúmen Wiki** — wiki para jogadores de uma campanha de TTRPG (Daggerheart) ambientada no cenário de dark fantasy "Domínio de Lúmen". Construída com Docusaurus 3.9 (TypeScript, React 19). Todo conteúdo deve ser escrito em **português do Brasil (pt-BR)**.

## Commands

- `npm start` — local dev server with hot reload
- `npm run build` — production build to `build/`
- `npm run serve` — serve the production build locally
- `npm run typecheck` — run TypeScript type checking (`tsc`)
- `npm run clear` — clear Docusaurus cache (useful when builds behave unexpectedly)

## Architecture

- **Docusaurus config**: `docusaurus.config.ts` — site metadata, navbar, footer, presets, theme
- **Sidebar config**: `sidebars.ts` — auto-generated from `docs/` folder structure
- **Docs content**: `docs/` — Markdown/MDX files, organized into folders with `_category_.json` for ordering
- **Blog content**: `blog/` — recaps de sessão (resumos do que aconteceu em cada sessão de jogo)
- **Custom pages**: `src/pages/` — React (TSX) or Markdown pages routed by filename
- **Components**: `src/components/` — reusable React components
- **Styles**: `src/css/custom.css` — global CSS variable overrides (Infima theming)
- **Static assets**: `static/` — images, favicon, served at site root

## Key Details

- Node >= 20 required
- Uses `@docusaurus/preset-classic` with docs, blog, and theme plugins
- `onBrokenLinks: 'throw'` — build fails on broken internal links
- `future.v4: true` — forward-compatibility flag for Docusaurus v4
- Sidebar uses `autogenerated` mode — folder structure in `docs/` determines navigation
- CSS theming uses Infima variables in `src/css/custom.css`

## Propósito & Público

Este wiki é destinado aos **jogadores** da campanha. Serve como referência para consulta sobre o cenário, NPCs encontrados, facções conhecidas, locais visitados e lore público do mundo. **Não é uma ferramenta do Mestre (DM).**

## Restrições de Conteúdo (CRÍTICO)

- **NUNCA** incluir segredos do plot, motivações ocultas de NPCs, ou informações que os jogadores ainda não descobriram em jogo.
- **NUNCA** revelar mecânicas internas de artefatos (propriedades ocultas, origens secretas).
- **NUNCA** incluir informações de facções que os jogadores não conhecem ainda.
- **NUNCA** incluir spoilers sobre eventos futuros, planos de vilões, ou conspirações não reveladas.
- Quando em dúvida, **omita**. É preferível ter menos informação do que vazar um segredo.
- Informação que os jogadores **sabem por backstory dos PCs** (ex.: Lyos sabe sobre Casa Rise, Mez sabe sobre as fábricas de Urna) pode ser incluída de forma superficial, sem revelar conexões ocultas.

## Abordagem de Conteúdo — Híbrida (Enciclopédica + Voz Narrativa)

Cada página segue a estrutura:
1. **Epígrafe/citação in-world** — uma frase evocativa no tom do cenário (sombrio, político, dramático) antes do conteúdo.
2. **Conteúdo objetivo** — informação clara, organizada em seções, prática para consulta rápida.
3. **Cross-links** — referências cruzadas entre páginas relacionadas (locais ↔ NPCs ↔ facções).

O tom das epígrafes deve ser: sombrio, político, íntimo e dramático. Sem humor, sem leveza, sem tom realista moderno.

## Blog — Recaps de Sessão

- Cada sessão jogada gera um post no `blog/`.
- Formato: data da sessão como título, resumo narrativo do que aconteceu.
- Tom: narrativo mas acessível — os jogadores devem conseguir relembrar os eventos.
- Pode incluir: eventos-chave, NPCs encontrados, decisões tomadas, consequências visíveis.
- **NÃO** incluir: consequências ocultas, avanços de relógios do DM, planos de facções.

## Locais Visitados vs. Não Visitados

**Visitados (conteúdo detalhado):**
- Encruzilhada — cidade-base, múltiplos distritos explorados
- Ponte das Correntes — travessia completa
- Sino — chegada recente

**Não visitados (conteúdo breve, contextualizador):**
- Véu, Urna, Trono de Deus, Pavio — os jogadores sabem que existem por backstory e rumores, mas nunca estiveram lá.

## Tema Visual

- Dark fantasy — cores escuras, tons de âmbar, cinza e dourado.
- A paleta de cores do Infima/CSS deve refletir o tom sombrio do cenário.
- Evitar cores vibrantes ou alegres. Preferir: preto, cinza-escuro, dourado-âmbar, branco-osso.
